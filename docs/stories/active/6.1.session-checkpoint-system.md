## Story

**As a** desenvolvedor que trabalha em múltiplas sessões (inclusive desligando o PC sem aviso),
**I want** um sistema automático de checkpoint que registra todo progresso do projeto,
**so that** qualquer agente em qualquer sessão futura saiba exatamente onde estávamos e continue a partir dali.

## Dev Notes

### Filosofia

O checkpoint é um "save point" do projeto. Como num RPG — se o jogador desligar o console, ao religar ele sabe exatamente onde estava. O checkpoint deve ser:
- **Completo:** tudo que um agente precisa saber para continuar
- **Conciso:** sem redundância, sem texto desnecessário
- **Atualizado:** sempre reflete o estado real do projeto
- **Acionável:** lista próximos passos claros

### Quando Atualizar

- Após cada story ser completada (Ready for Review / Done)
- Após criação de documentos novos (PRD, Architecture, Story)
- Após decisões arquiteturais (ADRs)
- Após push/merge para remote
- Ao final de cada sessão de trabalho

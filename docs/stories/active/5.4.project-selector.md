## Story

**As a** usuário com múltiplos projetos LMAS,
**I want** selecionar qual projeto carregar antes de entrar no Construct,
**so that** todos os dados (agentes, stories, métricas) sejam escopados ao projeto selecionado, sem misturar informações.

## Acceptance Criteria

1. [x] Após o typewriter da LoginScene, aparece a tela de seleção de projeto (ProjectSelectScene)
2. [x] Server descobre projetos LMAS automaticamente via scan de diretórios com `.lmas-core/` ou `core-config.yaml`
3. [x] API endpoint `GET /api/projects` retorna lista de projetos detectados (name, path, lastActivity)
4. [x] Lista de projetos exibida em estilo Matrix (terminal verde, fonte monospace, hover com glow)
5. [x] Ao selecionar um projeto, as pílulas aparecem: 🔴 Vermelha (entrar) / 🔵 Azul (voltar para seleção)
6. [x] Pílula vermelha → transição glitch → ConstructScene com dados do projeto selecionado
7. [x] Pílula azul → volta para a lista de projetos (ProjectSelectScene)
8. [x] Server recebe projeto selecionado via `POST /api/project/select` e reinicializa StateManager com novo projectRoot
9. [x] SSE broadcast, DataReader e FileWatcher passam a operar escopados ao projeto selecionado
10. [x] Se apenas 1 projeto detectado, auto-seleciona e vai direto para as pílulas (skip lista)
11. [x] Se nenhum projeto detectado, mostra mensagem: "Nenhum projeto LMAS encontrado" com instrução

## Dev Notes

### Descoberta de Projetos

O scan busca diretórios que contenham `.lmas-core/` ou `core-config.yaml`. Paths de busca:
1. Env var `LMAS_PROJECTS_PATH` (lista separada por `;` no Windows, `:` no Linux)
2. Diretório pai do servidor (workspace padrão)
3. Home do usuário (scan shallow, apenas 1 nível)

### Impacto em Módulos Existentes

- **`index.ts`** — `PROJECT_ROOT` deixa de ser hardcoded, inicializado via `/api/project/select`
- **`StateManager`** — Já aceita `projectRoot` no constructor (sem mudança de interface)
- **`DataReader`** — Já aceita `projectRoot` no constructor (sem mudança de interface)
- **`FileWatcher`** — Já aceita `projectRoot` no constructor (sem mudança de interface)
- **`LoginScene`** — Pílulas movidas para ProjectSelectScene; LoginScene só faz typewriter → transição
- **`api.ts`** — Novas rotas: `GET /api/projects`, `POST /api/project/select`

### Segurança

- Validar que projectPath passado no POST é um diretório real com `.lmas-core/`
- Não permitir path traversal (`../../../etc/passwd`)
- Sanitizar paths antes de usar no filesystem
